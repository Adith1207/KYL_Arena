# KYL Arena AI Development Guidelines

## Project Overview

KYL Arena is a mobile-first challenge management platform for fitness communities.

The platform allows communities to:

- Create fitness challenges
- Connect Strava accounts
- Track participant progress
- Generate leaderboards
- Manage competitions

Initial focus:

- Cycling
- Running
- Walking

---

## Core Principles

1. Mobile-first design.
2. Simplicity over complexity.
3. Build only what is required today.
4. Avoid premature optimization.
5. Every feature must improve community engagement.

---

## Tech Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- ShadCN UI
- Supabase
- Strava API

---

## Coding Standards

- Prefer server components when possible.
- Use TypeScript strictly.
- Avoid unnecessary dependencies.
- Keep components small and reusable.
- No inline styles.
- Use Tailwind CSS.

---

## UI Guidelines

Design should feel:

- Clean
- Modern
- Athletic
- Community-driven

Prioritize mobile screens before desktop layouts.

---

## Folder Philosophy

Create folders only when required.

Do not generate large unused architectures.
